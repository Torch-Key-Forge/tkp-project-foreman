# Project Foreman Evidence Boundary — Demo Insert

## Placement

Place this segment after the Authority Ledger and before the Continuation Brief.

## Duration

Target: 12–15 seconds.

## Narration

> Project Foreman packets remain evidence-bound when moved into a new environment. Canonical project evidence stays separate from provisional review, unresolved state, and unavailable sources. If support is missing, the claim is not established—no assistant authority promotion and no completion inference.

## On-screen sequence

1. Open the **Evidence Boundary** tab.
2. Hold on **No source, no claim.**
3. Show the unsupported claim test:
   - Question: `Did the client approve production deployment?`
   - Result: `NOT_ESTABLISHED`
   - Response: `This claim is not established in the imported project evidence.`
4. Keep the control lines visible:
   - Assistant authority promotion: `NONE`
   - Command completion without evidence: `NOT_INFERRED`
   - Candidate disposition: `HELD_FOR_REVIEW`
   - Promotion: `OPERATOR_REVIEW_REQUIRED`

## Claim boundary

This segment demonstrates deterministic packet policy and fixture behavior. It does not claim live access to Google Drive, email, GitHub, or any external workspace.
